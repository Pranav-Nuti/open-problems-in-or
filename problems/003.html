Problem #3

Computational Complexity of Multi-Stage Stochastic Programming

Author Andy Philpott, Alexander Shapiro
Difficulty Hard
Chance of Being Open High

Preliminary Notation

Consider a \(T\)-stage stochastic linear program:

\[ \min_{x_1} \; c_1^\top x_1 + \mathbb{E}_{\xi_2}\left[ \min_{x_2} c_2(\xi_2)^\top x_2 + \cdots + \mathbb{E}_{\xi_T|\xi_{2:T-1}}\left[ \min_{x_T} c_T(\xi_T)^\top x_T \right] \right] \]

subject to:

\[ A_1 x_1 = b_1, \quad T_t(\xi_t) x_{t-1} + W_t(\xi_t) x_t = h_t(\xi_t), \quad x_t \geq 0 \]

where \( \xi_t \) denotes the random data at stage \(t\), and \( x_t \) is the decision at stage \(t\) (possibly depending on \( \xi_2, \ldots, \xi_t \)). The scenario tree has \( S \) scenarios; if \( S \) is finite, the problem becomes a large linear program.

Problem Statement

Open Problem: Multi-stage stochastic linear programming is known to be PSPACE-hard in the number of stages (Dye et al., 2003). For two-stage stochastic programs, the problem is NP-hard when the distribution is discrete with a polynomial number of scenarios.

Key questions that remain open:

  1. For a fixed number of stages \(T\), what is the precise complexity of solving multi-stage stochastic LPs when the scenario tree has \(n\) nodes? Is there an algorithm with running time polynomial in \(n\) (and the input size) when \(T\) is fixed?
  2. Under what conditions on the underlying process (e.g., stagewise independence, finitely supported distributions) can we obtain tractable approximations with provable guarantees?
  3. What is the complexity of computing an \(\varepsilon\)-optimal solution when the uncertainty is continuous (e.g., driven by Brownian motion)?
